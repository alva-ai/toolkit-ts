### Task 8: Add `writeConfig` function (TDD)

**Files:** `src/cli/config.ts`, `test/config.test.ts`

**What to do:** Add a `writeConfig()` function to `config.ts`. It accepts
`{ apiKey: string, baseUrl?: string }` and writes JSON to the config path.
Uses dependency injection like `loadConfig` for testability.

Behavior:

- Resolve config path: `$XDG_CONFIG_HOME/alva/config.json` or
  `~/.config/alva/config.json`
- Create parent directory with `mkdir({ recursive: true })`
- If config file exists, read and merge (preserve fields not being set)
- Write JSON with `{ mode: 0o600 }` for file permissions
- Return the written config object

**Steps:**

- [ ] Write failing tests first:
  - writeConfig happy path (writes file, returns config)
  - writeConfig creates parent directories
  - writeConfig sets file permissions to 0o600
  - writeConfig merges with existing config
  - writeConfig handles mkdir failure
  - writeConfig handles writeFile failure
- [ ] Run tests, verify they fail for the right reason
- [ ] Implement `writeConfig()` in `config.ts`
- [ ] Run tests, verify all pass
- [ ] Run lint: `npm run lint`
